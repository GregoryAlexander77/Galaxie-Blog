BlogCFC Update

Apr 22

posted by Raymond Camden in BlogCFC | 1 Comments

Some changes were just checked in. The big change is that delayed entries now work. Not only will they show up at the right time now, but they will also send email to the subscribers.

I cannot describe how much of a pain this was, and it came as a total surprise. It's funny how some features are big in impact but are easy to code. I don't think many people will use this feature, but it was a real pain in the rear to get it right.

It handles the delayed publishing by simply creating a one time event. When the event is run it mails subscribers and it removes the scheduled task.

I will be posting a test after this entry.

I also did a few other small things behind the scenes, but nothing worth mentioning.

So I think Scott and I got a lot done this week. My next big feature - Related Entries. Then I hope to do a push to my live blog to get it using the new features. Luckily someone has already done the work for Related Entries, I just need to review and integrate. It is a significant feature though so that will be the only thing for next week.

After that, I think I may move this application to beta status. I'll update the install docs and sql scripts and try to get more people using the application so we can flesh out more bugs.

Entries with a thousand words

Apr 19

posted by Raymond Camden in BlogCFC | 3 Comments

Roger (I believe) suggested something over at my main blog, and that was an easier way to handle images. I love using images in blog posts, but hate the trouble of inserting the HTML and using FTP to upload the file.

Now BlogCFC will notice an enclosure that is either a JPG, GIF, or PNG. When it finds it, it will automatically add it to the top left of an entry.

Note that Preview isn't working with this yet.

Big thanks to Zoid for getting the CSS right for hits. Today's update brought to you by some old school "My Life with the Thrill Kill Kult."

Entries display changes

Apr 15

posted by Raymond Camden in BlogCFC | 2 Comments

I changed the entries display in the admin today. It was quite a bit slow for my blog, which is probably not the norm with 1.1k entries. However, by making this change, it went from taking 2 seconds or so to run to less then half a second. I also had a QofQ bug as well.

With this change, you don't see the comment count or categories on the Entries page, but I figure the author probably doesn't really need those fields anyway, and the speed difference was really nice.

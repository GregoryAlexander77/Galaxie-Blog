Category Bug

May 29

posted by Raymond Camden in BlogCFC | 3 Comments

There is a bug if you give a new category a name with a comma in it. The bug will crop up on the front page. My code assumes a list of category names, and obviously a category name with a comma in it will screw things up. I fixed this on another blog, but this is not in source control yet. (And you will see why when you read the code):

view plain copy to clipboard print about
  ColdFISH is developed by Jason Delmore. Source code and license information available at coldfish.riaforge.org
 <cfloop index="x" from=1 to="#listLen(categoryIDs)#">
    <a href="#application.blog.makeCategoryLink(listGetAt(categoryIDs,x))#">#application.blog.getCategory( listGetAt(categoryIDs,x)).categoryname#</a><cfif x is not listLen(categoryIDs)>,</cfif>
</cfloop> 1<cfloop index="x" from=1 to="#listLen(categoryIDs)#">
2    <a href="#application.blog.makeCategoryLink(listGetAt(categoryIDs,x))#">#application.blog.getCategory( listGetAt(categoryIDs,x)).categoryname#</a><cfif x is not listLen(categoryIDs)>,</cfif>
3</cfloop>

As you can see, on each call I'm fetching the category for ID. This is overkill on a page with lots of categories. What I'll probably do is change getEntries to make the categorynames field store a struct or some such.

GetCategories change (need Access tester)

May 10

posted by Raymond Camden in BlogCFC | 3 Comments

I made a change to getCategories today so that it would return categories with 0 entries. That's a rare condition probably, but I think all categories should always be returned, even if they have 0 entries.

The SQL was tricky. With help from a friend I got it working in SQL Server using one query. However, the code didn't work quite right in MySQL. So for MySQL and Access, I'm doing a hack. Since the pod itself is cached, I don't think it is a huge big deal. If someone using Access could give it a try, I'd appreciate it.

Oh, and if you feel up to it. Check the code for getCategories. In the MSSQL section, you will see code in there for MySQL. It isn't quite working. Instead of returning all categories, it only returns categories with 0 entries. If you can get it working, let me know.

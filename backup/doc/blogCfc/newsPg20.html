Slideshow Spec

Jul 15

posted by Raymond Camden in BlogCFC | 10 Comments

I'm considering a slide show feature for BlogCFC 5.2. It would work like so:

The admin would let you add/edit/delete slideshows. A slide show is nothing more than a title, and a folder name. So the title may be "Pictures of my foot" and the folder would be named foot. Slideshows would be located at yourblog/images/slideshows/XXXX where XXXX is the folder name. The admin would let you upload images. I wouldn't do any fancy resizing or anything, just basic file uploading,

At that point, you will have a URL (a short one) for your slide show and you can share it. It need not be tied to a blog entry. If you want though, you can do:

<slideshow name="foot"> to embed a link to a slide show in a blog entry/page. It embeds just the link of course.

Thoughts? As you can tell - I'm really looking at ways to expand the blog to new areas.

BlogCFC 5.1 Alpha 5

Jul 11

posted by Raymond Camden in BlogCFC | 7 Comments

A few changes:

  • I added caching to the RSS page. This only works for the main cache display, but since this is probably the most hit URL on your blog, it should help quite a bit. Don't ask me why I didn't do this earlier.
  • Ok you freaks. The entry form in the admin now calls out to textarea.cfm, a very simple custom tag. If you want to use a rich text editor, you would simply modify this custom tag. This is easier than editing the entry.cfm page as you don't have to worry about breaking anything else in the editor. So. There. I did it. If I hear one more person complain about rich text editor support I'm going to switch to classic ASP!
  • There is an issue with IE, Gravatars, and one line comments. I have my CSS flunky partner Boyzoid working on it. He says he knows the issue, but not the solution. But he is working on it. Not "Indiana Jones" type "top people" working on it, but really working on it.

So - I'm almost done with 5.1. I will work on the documentation this week, but outside of that, the only feature left is the AJAX-ification of the entry editor, which, admittedly, will only help those of you with lots of blog entries.

BlogCFC 5.1 Alpha 3

Jul 9

posted by Raymond Camden in BlogCFC | 7 Comments

Once again, I'm just plain disappointed in you folks. Sure, I got some feedback, but not enough. I give, and give, and give, and you guys just take, and take and take. Well fine then. I'll release another alpha.

This version adds textblock support. What's a textblock? A block of text. I know, surprising, eh? A textblock is a useful way to store information you edit often. So for example, imagine your local user group uses the blog. You have a "Next Meeting" pod. Normally you edit the file yourself. But with textblock support, you could edit the data via the BlogCFC administrator.

To use the content in your pod, you would simply do:

BlogCFC 5.1 Alpha 2

Jul 8

posted by Raymond Camden in BlogCFC | 13 Comments

So, since no one commented on the release last night, I got mad and decided, "Screw these guys, I'll just add more features!" Just kidding of course.

Today's release adds a major new feature, Pages. The pages feature lets you add any number of pages to your site. This would be useful for things like, "About Me" pages, "Contact Me" pages, etc. You can easily edit the pages from the administrator and the application will automatically generate a Search Engine Safe URL for your page.

You can get this feature by updating from Subversion. Please check the release notes for db change information. This feature is NOT in the documentation.

What's next?

I'm giving up. Seriously. I can't take it anymore. I'm going to take the code that let's you edit blog entries, and abstract out the simple text area box. This will also be used in other areas of the admin. What's the point? You can take this tag and edit it to put your rich text editor in. You know, if you are into that kind of thing. Me - I think rich text editors are for liberal commie pinko ASP users, but.... it's obvious that I'm in the minority.

After that I'll add textblock support and that may be it for 5.1.

Sep 2

posted by Raymond Camden in BlogCFC | 3 Comments

5.2 Status Update

Sep 1

posted by Raymond Camden in BlogCFC | 8 Comments

I decided that the next refresh of 5.2 would include ATOM support, which Rob Rohan graciously released to the world. It took me all of 5 minutes to get his code working, but then I discovered a major problem. This is with the ATOM spec, not Rob's code. The problem is that the spec does not include support for getting blog categories. To me - that is incredibly dumb, but, hey, what do I know.

Because of that - I have decided not to add Atom support. Obviously folks can download Rob's code if they want it.

I decided to take another look at XML-RPC and the Metaweblog API. The first thing I found is that it is near impossible to get a decent spec. Sure you can find docs - but lord forbid they provide a simple list of calls and results. That really ticked me off. I mean, to me, it is an obvious thing I'd provide in the documentation.

Another problem is that it is really hard to test. Luckily I can use ServiceCapture to see the XML going back and forth.

Last night I downloaded Ecto and started to play and I've made some progress. I'm hoping to have this done in a week, and then I'd like folks to test. Luckily this whole aspect of the blog is separated, so if things don't work out, I can hold it off till 5.3 if need be.

How many people would really like this feature?

BlogCFC 5.2 Alpha 2

Aug 25

posted by Raymond Camden in BlogCFC | 25 Comments

This is a major update and it is all thanks to Deanna Schneider and Scott Stroz. The changes are:

  • BlogCFC now has support for Oracle.
  • If your DSN requires a username and password, you can now supply it in the INI file. You no longer need to edit all the cfquery tags. (The settings editor in the admin hasn't been updated yet.)
  • New add comment design and send.cfm cleaned up to be more CSS, less table.
  • Documentation updated (partially, covers slide shws)

What is left for 5.2? I want to polish off the rendering enginge and supply a few more rendering rules. After that - well - I may just ship. I'm also considering going ahead and adding in admin support for the slide shows.

Two Spam Changes

Aug 21

posted by Raymond Camden in BlogCFC | 3 Comments

In the last release, I updated send.cfm to use captcha. A user reported an error where the form field didn't exist (most likely due to a spammer using their own version of the form). This has been updated in SVN.

Andy Jarret reported that by changing the form field name in the Subscribe pod, he was able to block most spammers from signing up (since spammers have the brains of small field peas). I used his suggestion and made the field dynamic. This will let you rename it to cucumber if you wish.

First BlogCFC alpha checked in

Aug 20

posted by Raymond Camden in BlogCFC | 0 Comments

The first alpha of BlogCFC 5.2 is now checked in. Included in this build are the following changes:

  • Use of rb() instead of application.resourceBundle.getResource. This is a non-event for most people, but it makes life a heck of a lot easier for me.
  • CAPTHA support for the Send form. This will help prevent folks from using send.cfm as a spam device.
  • Use of icons for the menus below entries.
  • Display of Views by each entry (thanks Rob!)
  • Slideshow support in (not documented)
  • Stats page: Subscribers stats will only count verified users.
  • Render support in with one render tool, include. No documentation exists. Warning - This feature is definitely version 0, so use at your own risk. Also note that I'll be tweaking it a bit more in the next alpha.

Plans for the next alpha: Increase Render engine support. Tag cloud support. Confirm date fix issue for D/M/YYYY users.

Survey Responses

Oct 25

posted by Raymond Camden in BlogCFC | 7 Comments

Thank you to everyone who filled out the survey. I'm going to keep it open for a bit longer, but I thought I'd share the results now. I have a second survey launching tonight, and for those of you who helped spread the word last time, I'd love it if you could do it again. I've included both a PDF and an Excel sheet to this blog entry if you want the raw data. Frankly, I don't know how to use Excel enough to do fancy stuff. I viewed the HTML report on my Soundings admin and here is some general things I noticed.

A good chunk of you have been using BlogCFC for more than 2 years. WOW. I hate to say it - but I'm still surprised people actually use my software. I know people do - but to see the amount of using it for 2+ years, and then 1-2 years, was amazing. Thanks for sticking around. ;)

Not surprisingly - most of you are hosting BlogCFC on Windows. I don't blame you. I love my Mac - but when it comes to a web server, I really find Windows easier to set up.

Most of you are on ColdFusion 7 and 8. Adobe will love that. Only 5 of you are on 6. Good. See my second survey for news on that topic.

Most people use SQL Server and MySQL. I may drop support for MS Access and add support for Derby. We shall see. If I do - it will be towards the end of the GoldenSnitch process.

When it comes to version, most folks were in 5.X, with a good portion at 5.8.001 (last rev before 5.9). Obviously the fact the more people aren't at 5.9 is indicative of the upgrade issues we have in general. (Note, anything wrong with BlogCFC is a 'we' problem. Anything good is all me. :)

There seemed to be no major 'other' blog except perhaps Mango. Mango does look nice and I consider it my major "competition" for v6.

When it comes to features - I was surprised by the amount of folks who didn't know about textblocks, slideshows, and pages. I think for v6 I want to improve the docs some and spend some time focusing on the 'non-blog' type stuff. Personally I think it is very powerful. Shoot, the pages stuff could almost be considered a minor CMS! (Well, not really.) I was surprised by how few people use the Slide Shows. Any comments on that? Of course, I tend to use my slide shows with pictures of my kids, so it could be a parent thing. But is the feature too hard to use? Not good enough?

Thanks again folks.

You mean people use this?

Oct 11

posted by Raymond Camden in BlogCFC | 8 Comments

I just got off the phone with a client who is having a bit of an issue with BlogCFC. During the call I mentioned that they should change a line in Application.cfm. They asked if I could do it. I said - well - no - I can't, not without FTP access.

Then I got real embarrassed. Turns out - they had done this themselves using the File Manager tool I had built. They used it to download Application.cfm, edit, and then reupload.

I had never imagined someone would do that. I had mainly built the File Manager to let you upload resources and stuff like that.

Release Notes for 5.9

Oct 9

posted by Raymond Camden in BlogCFC | 7 Comments

Just thought I'd share the release notes for 5.9 - some nice gems in there. I hope to release this by end of the week or so.

/client/addcomment.cfm - Default http:// in the URL. For times, use BlogCFC's offset time. /client/admin/index.cfm - Two small text changes.

/client/admin/pod.cfm, podform.cfm, pods.cfm - Scott P's changes to allow you to edit pods

/client/admin/settings.cfm - Increase maxlength on users field.

/client/admin/spryproxy.cfm - specify UTF-8

/client/includes/admin.css - made some form fields 100% wide. To me - makes editing easier.

/client/includes/pod.cfm - add a timeout to the CFHTTP

/client/includes/style.css - few small changes. Only one I remember is the Mac font fix.

/client/rss.cfm - specify UTF-8

/client/search.cfm - remove debug code, fix cgi code

/client/tags/getmode.cfm - Don't restrict to last 30 days, and handle 'large url var' issue.

/client/tags/scopecache.cfm - Fix the damn 'dependancies' error that shouldn't exist.

/client/install/mysql.sql - updated install sql

/org/camden/blog/ping.cfc, ping7.cfc, ping8.cfc - New ping support. In CF8, we use CFTHREAD to fire the pings.

/org/camden/blog/blog.cfc - Detect if cf8, load right ping, fix in generateRSS for format of times, run code blocks before Render, allow for verify via email, getNumberUnmoderated fixed to get # for blog

/org/camden/blog/render/render.cfm - Trim results, this was someone else's idea - I believe Scott P.

/client/admin/entry.cfm - Fix a bug with dates (again), make the table wider.

/client/includes/spry/* - Minimized, 1.6 versions

/client/admin/subscribers.cfm - Let admin verify people

/client/tags/datatable.cfm - support for above

/client/admin/entries.cfm - On delete, clear cache

/client/admin/settings.cfm - Handle cases where a login attempt sets username/password in the form scope

/client/stats.cfm - Added an average views per entry value

/client/includes/main.properties - For above

/client/includes/main_en_US.properties - For above (and added stuff for contact form)

/org/camden/blog/ping7.cfc - wrap the cfhttp with a try/catch. Only done for bare pings.

/org/camden/blog/blog.cfc - Use Brian Kotek's code to allow IP wildcards for spam blocking.

Removed: sex, lesbian, porn, from the spam block list. I felt these words were a bit too vague to be included.

/client/contact.cfm - New contact form

/client/tags/layout.cfm - new URLs for me, link to contact.
